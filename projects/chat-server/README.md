
# I/O Multiplexing 기반 멀티 클라이언트 채팅 서버
## Overview

C / Linux 환경에서 TCP/IP Socket 기반 멀티 클라이언트 채팅 서버를 구현한 프로젝트입니다.
서버는 `select()`와 `fd_set`을 사용하여 서버 소켓과 여러 클라이언트 소켓을 동시에 감시하고, 신규 접속과 메시지 수신을 구분하여 처리합니다.

접속한 클라이언트는 닉네임을 등록하고, 서버는 닉네임 중복 여부를 검사합니다.
특정 클라이언트가 메시지를 보내면 서버 기준 타임스탬프와 발신자 이름을 붙여 모든 클라이언트에게 브로드캐스트합니다.

---

## Tech Stack

* C
* Linux
* TCP/IP Socket Programming
* `socket()`, `bind()`, `listen()`, `accept()`
* `select()`, `fd_set`
* Linked List
* `time.h`

---

## Features

### 1. TCP 서버 구조 구현

* `socket()`을 사용하여 서버 소켓 생성
* `bind()`를 사용하여 서버 주소와 포트 바인딩
* `listen()`을 사용하여 클라이언트 접속 대기
* `accept()`를 사용하여 신규 클라이언트 접속 처리

### 2. select() 기반 I/O Multiplexing

* 단일 스레드 환경에서 여러 클라이언트의 입력 이벤트 감시
* 서버 소켓과 클라이언트 소켓을 `fd_set`에 등록
* 신규 접속 이벤트와 기존 클라이언트 메시지 수신 이벤트를 구분하여 처리

### 3. 연결 리스트 기반 클라이언트 관리

* 접속 중인 클라이언트의 소켓 번호와 닉네임을 연결 리스트로 관리
* 신규 접속 시 클라이언트 정보를 리스트에 추가
* 접속 종료 시 해당 클라이언트 정보를 리스트에서 제거
* 현재 접속 중인 클라이언트 상태를 서버가 중앙에서 관리

### 4. 닉네임 등록 및 중복 검증

* 클라이언트 최초 접속 시 닉네임 등록
* 이미 사용 중인 닉네임으로 접속을 시도하면 서버가 등록 거부
* 닉네임 중복이 없는 경우에만 채팅 참여 허용

### 5. 메시지 브로드캐스트

* 특정 클라이언트가 메시지를 보내면 서버가 이를 수신
* 서버는 발신자 이름과 메시지를 모든 클라이언트에게 전달
* 여러 클라이언트가 동시에 대화할 수 있도록 구현

### 6. 타임스탬프 출력

* `time.h`를 사용하여 서버 기준 현재 시간 생성
* `strftime()`을 사용하여 `[YYYY-MM-DD HH:MM:SS]` 형식의 시간 문자열 생성
* 각 메시지 앞에 시간과 발신자 이름을 함께 출력

## Architecture

```text
Client 1 ─┐
Client 2 ─┼── TCP Chat Server ── select() / fd_set
Client 3 ─┘

Server
├─ socket() / bind() / listen()
├─ accept() : 신규 클라이언트 접속 처리
├─ select() : 서버 소켓과 클라이언트 소켓 감시
├─ Linked List : 클라이언트 소켓 번호와 닉네임 관리
├─ Nickname Check : 닉네임 중복 검증
├─ time.h : 서버 기준 타임스탬프 생성
└─ Broadcast : 모든 클라이언트에게 메시지 전달
```

---

## Client Flow

```text
프로그램 실행
→ 서버 접속 요청
→ 서버의 접속 승인 확인
→ 닉네임 등록 요청
→ 닉네임 중복 여부 확인
→ 메시지 송수신
→ EOF 입력 또는 연결 종료 시 종료
```

---

## Server Flow

```text
서버 실행
→ socket / bind / listen
→ select()로 입력 이벤트 감시
→ 신규 접속 발생 시 accept()
→ 클라이언트 닉네임 등록 및 중복 검증
→ 메시지 수신 시 전체 클라이언트에게 브로드캐스트
→ 클라이언트 종료 시 연결 리스트에서 제거
```
## How to Run

이 프로젝트는 수업 과제 환경에서 지정된 포트를 기준으로 구현한 TCP 채팅 서버입니다.
서버 프로그램을 먼저 실행한 뒤, 별도의 터미널에서 클라이언트 프로그램을 실행하여 서버에 접속하는 방식으로 동작합니다.

gcc server.c -o server
gcc client.c -o client

서버 실행:
```
./server
```

클라이언트 실행:
```
./client
```
여러 개의 클라이언트 프로그램을 실행하면 닉네임 등록, 닉네임 중복 검증, 메시지 브로드캐스트, 접속 종료 처리를 확인할 수 있습니다.

※ 포트 번호와 접속 환경은 수업 과제에서 사용한 환경을 기준으로 작성되었습니다. 다른 환경에서 실행할 경우 포트 번호, 서버 주소, 방화벽 설정에 따라 수정이 필요할 수 있습니다.

## What I Learned

이 프로젝트를 통해 TCP/IP 기반 클라이언트-서버 통신 구조와 `select()`를 활용한 I/O Multiplexing 방식을 학습했습니다.
또한 서버가 여러 클라이언트의 접속 상태를 중앙에서 관리하고, 이벤트가 발생한 소켓을 구분하여 처리하는 흐름을 경험했습니다.

게임 서버 개발에서도 접속 세션 관리, 유저 상태 관리, 닉네임 또는 ID 중복 검증, 메시지 전달, 접속 종료 처리와 같은 기능이 중요하다고 생각합니다.
이 프로젝트는 이러한 서버 개발의 기본 개념을 C와 Linux 환경에서 로우레벨로 경험한 프로젝트입니다.
