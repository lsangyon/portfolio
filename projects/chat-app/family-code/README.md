# C# TCP 실시간 채팅 서버 - 가족코드 인증과 다중 클라이언트 세션 관리

기존 C# TCP 채팅 서버를 확장하여, 다중 클라이언트 접속 처리와 닉네임 기반 세션 관리에 가족코드 인증 흐름을 추가한 프로젝트입니다. TcpListener/TcpClient 기반 통신 구조 위에서 클라이언트 입장 전 인증 단계를 분리하고, 닉네임 중복 검사, 메시지 브로드캐스트, 명령어 처리, 연결 종료 처리를 구현했습니다. 또한 이후 Unity/모바일 클라이언트로 확장하기 쉽도록 주소, 포트, 닉네임, 가족코드를 항목별로 입력하는 흐름으로 변경했습니다.

이전 버전에서는 닉네임을 입력하면 바로 서버에 접속할 수 있었지만, 이번 버전에서는 가족코드를 알고 있는 사용자만 채팅방에 들어올 수 있도록 수정했습니다.

## 주요 기능

### 서버
- `TcpListener` 기반 TCP 서버 실행
- 여러 클라이언트의 동시 접속 처리
- 클라이언트별 닉네임/소켓 정보 관리
- 가족코드 인증 후 입장 허용
- 닉네임 중복 검사
- 채팅 메시지 전체 브로드캐스트
- `/nick` 명령어를 통한 닉네임 변경
- `/quit` 명령어 및 연결 종료 처리

### 클라이언트
- 서버 주소, 포트, 닉네임, 가족코드 항목별 입력
- 서버 인증 결과에 따른 입장/종료 처리
- 콘솔 기반 메시지 송수신

## 구조

```text
Client
  └─ 주소/포트/닉네임/가족코드 입력
        ↓
TCP Server
  ├─ 가족코드 인증
  ├─ 닉네임 중복 검사
  ├─ 클라이언트 세션 관리
  ├─ 명령어 처리 (/nick, /quit)
  └─ 메시지 브로드캐스트

## 접속 처리 흐름

1. 서버가 지정 포트에서 TCP 접속을 대기합니다.
2. 클라이언트가 서버 주소와 포트를 입력해 접속합니다.
3. 서버는 클라이언트에게 가족코드 입력을 요구합니다.
4. 클라이언트가 가족코드를 전송합니다.
5. 서버는 가족코드가 올바른지 확인합니다.
6. 인증에 성공한 경우 닉네임 등록 단계로 이동합니다.
7. 서버는 닉네임 중복 여부를 검사합니다.
8. 닉네임 등록이 완료되면 클라이언트를 접속자 목록에 추가합니다.
9. 이후 수신한 메시지를 다른 클라이언트에게 브로드캐스트합니다.
10. 연결 종료 또는 `/quit` 입력 시 접속자 목록에서 제거하고 퇴장 메시지를 전송합니다.

## 가족코드 기능

클라이언트가 서버에 접속할 때 가족코드를 함께 입력하도록 했습니다.

서버는 클라이언트가 보낸 가족코드를 확인하고, 코드가 맞는 경우에만 채팅방 입장을 허용합니다.
코드가 맞지 않으면 채팅방에 들어오지 못하고 연결이 종료됩니다.

이 기능은 단순한 공개 채팅방이 아니라, 가족처럼 정해진 사용자만 들어오는 채팅방을 만들기 위한 기본 입장 조건으로 구현했습니다.

## 입력 방식 변경

기존에는 클라이언트를 실행할 때 주소, 닉네임, 포트를 한 줄의 실행 인자로 넘기는 방식이었습니다.

예시:

```bash
dotnet run -- 127.0.0.1 alice 10140
```

이번 버전에서는 실행 후 필요한 값을 항목별로 입력하도록 변경했습니다.

예시:

```text
실행
주소
아이디
포트명
가족코드
```

이렇게 바꾼 이유는 이후 모바일 앱 화면으로 옮기기 쉽게 하기 위해서입니다.

모바일 환경에서는 사용자가 명령어를 직접 입력하는 방식보다는, 주소 입력칸, 아이디 입력칸, 포트 입력칸, 가족코드 입력칸처럼 항목별로 값을 넣는 방식이 더 자연스럽습니다.
그래서 콘솔 프로그램 단계에서도 입력 흐름을 모바일 화면 구조와 비슷하게 맞춰 보았습니다.

## 실행 요구사항

* .NET SDK 설치 필요

## 서버 실행 방법

```bash
cd ChatServer
dotnet run -- 10140
```

포트를 생략하면 기본값으로 `10140`번 포트를 사용합니다.

```bash
dotnet run
```

## 클라이언트 실행 방법

```bash
cd ChatClient
dotnet run
```

실행 후 화면에 표시되는 안내에 따라 주소, 아이디, 포트, 가족코드를 입력합니다.

같은 PC에서 테스트할 경우 주소는 다음과 같이 입력합니다.

```text
127.0.0.1
```

같은 Wi-Fi에 연결된 다른 기기에서 접속할 경우에는 서버 PC의 LAN IP 주소를 입력합니다.

예시:

```text
192.168.0.23
```

## 명령어

닉네임 변경:

```text
/nick 새닉네임
```

채팅 종료:

```text
/quit
```

## 포트폴리오 설명

이 버전에서는 단순히 닉네임을 가진 클라이언트가 접속하는 구조에서, 가족코드를 확인한 뒤 채팅방 입장을 허용하는 구조로 확장했습니다.

또한 실행 시 주소, 아이디, 포트 등을 명령행 인자로 한 번에 넘기던 방식을 항목별 입력 방식으로 바꾸었습니다.
이는 추후 모바일 앱으로 확장할 때 로그인/입장 화면을 구성하기 쉽게 하기 위한 변경입니다.

이번 작업에서는 채팅 기능 자체보다, 사용자가 채팅방에 들어오기 전의 입장 조건과 입력 흐름을 정리하는 데 초점을 두었습니다.
