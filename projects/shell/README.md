# C 기반 미니 UNIX Shell 구현

## Overview

C 언어와 Linux 환경에서 UNIX Shell의 일부 기능을 직접 구현한 프로젝트입니다.
사용자가 입력한 명령어를 파싱하고, 외부 프로그램 실행, 디렉토리 관리, 명령어 히스토리, 와일드카드, 프롬프트 변경, alias 기능 등을 처리하도록 구현했습니다.

이 프로젝트를 통해 Linux에서 명령어가 실행되는 흐름과 `fork()`, `execvp()`, `wait()`를 활용한 프로세스 생성 및 실행 구조를 학습했습니다. 또한 문자열 파싱, 동적 메모리 관리, 연결 리스트 기반 자료구조를 활용하여 Shell 기능을 확장했습니다.

## Features

### 1. 외부 명령어 실행

사용자가 입력한 명령어를 파싱한 뒤, 외부 프로그램을 실행할 수 있도록 구현했습니다.
자식 프로세스를 생성하고, 해당 프로세스에서 명령어를 실행하는 방식으로 동작합니다.

* `fork()`를 사용하여 자식 프로세스 생성
* `execvp()`를 사용하여 외부 명령어 실행
* Foreground 명령어의 경우 부모 프로세스가 `wait()`로 종료 대기
* Background 실행 명령어 처리 구조 구현

### 2. 디렉토리 관리 기능

Shell 내부 명령어로 디렉토리 이동과 디렉토리 스택 관리를 구현했습니다.

* `cd`: 현재 작업 디렉토리 변경
* `pushd`: 현재 디렉토리를 디렉토리 스택에 저장
* `dirs`: 저장된 디렉토리 스택 출력
* `popd`: 스택에 저장된 디렉토리로 이동

### 3. 명령어 히스토리 기능

사용자가 실행한 명령어를 저장하고, 이전 명령어를 다시 실행할 수 있도록 구현했습니다.

* `history`: 실행한 명령어 목록 출력
* `!!`: 직전에 실행한 명령어 재실행
* `!string`: 특정 문자열로 시작하는 최근 명령어 검색 후 실행
* 최대 32개의 명령어 기록 저장

### 4. 와일드카드 기능

명령어 인자에 `*`가 포함된 경우, 현재 디렉토리의 파일 목록으로 확장하는 기능을 구현했습니다.

예를 들어 현재 디렉토리에 여러 파일이 있을 때, `ls *` 명령어를 입력하면 현재 디렉토리의 파일 목록을 인자로 전달하도록 처리했습니다.

### 5. 프롬프트 변경 기능

사용자가 원하는 문자열로 Shell 프롬프트를 변경할 수 있도록 구현했습니다.

* 기본 프롬프트: `Command :`
* `prompt name` 입력 시 이후 프롬프트를 `name :` 형태로 변경

### 6. alias / unalias 기능

명령어 별칭을 등록하고 해제하는 기능을 구현했습니다.

* `alias command1 command2`: `command1`을 입력했을 때 `command2`로 치환하여 실행
* `unalias command1`: 등록된 alias 해제
* 연결 리스트를 활용하여 alias 목록 관리

### 7. rename 명령어

직접 추가한 명령어로, 파일 또는 디렉토리 이름을 변경하는 기능을 구현했습니다.

* `rename old_name new_name`
* C 표준 라이브러리의 `rename()` 함수 사용

## Implementation Highlights

- `fork()`와 `execvp()`를 사용하여 외부 명령어 실행 흐름을 구현했습니다.
- Foreground 실행의 경우 `wait()`를 사용하여 자식 프로세스 종료를 기다리도록 처리했습니다.
- 사용자의 입력 문자열을 명령어와 인자로 분리하는 파싱 로직을 구현했습니다.
- `cd`, `pushd`, `dirs`, `popd`를 내부 명령어로 구현하여 디렉토리 이동과 디렉토리 스택을 관리했습니다.
- `history`, `!!`, `!string` 기능을 통해 이전 명령어를 저장하고 재실행할 수 있도록 구현했습니다.
- 연결 리스트를 활용하여 alias 목록을 관리하고, 입력 명령어를 등록된 별칭으로 치환하도록 처리했습니다.
- `*` 문자가 포함된 인자를 현재 디렉토리의 파일 목록으로 확장하는 wildcard 기능을 구현했습니다.

## What I Learned

이 프로젝트를 통해 Shell이 단순히 명령어를 실행하는 프로그램이 아니라, 사용자 입력을 해석하고 프로세스를 생성하며 실행 결과를 관리하는 구조로 동작한다는 점을 이해했습니다.

특히 `fork()`와 `execvp()`를 활용하여 외부 명령어를 실행하는 과정, `wait()`를 통한 자식 프로세스 종료 대기, 문자열 파싱을 통한 명령어와 인자 분리, 연결 리스트를 활용한 alias 관리 등을 경험했습니다.

또한 `history`, `!!`, `!string`, `pushd`, `dirs`, `popd`와 같은 기능을 구현하면서, 사용자가 편리하게 명령어를 재사용하고 디렉토리 상태를 관리할 수 있도록 Shell 기능이 확장되는 방식을 배웠습니다.
